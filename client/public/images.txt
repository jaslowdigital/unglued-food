## ALT TEXT REQUIREMENTS

### Alt Text Best Practices
- Describe image content accurately in 125 characters or less
- Include primary keyword when relevant and natural
- Avoid keyword stuffing or repetitive phrases
- Be specific rather than generic
- Don't start with "image of" or "picture of"
- Include brand names when products are shown
- Describe actions happening in the image
- Mention colors only when relevant to understanding

### Alt Text Examples
**Good Alt Text Examples:**
- "Gluten-free chocolate chip cookies on cooling rack"
- "Slicing homemade gluten-free sourdough bread"
- "Mediterranean quinoa bowl with chickpeas and feta"
- "Organic Harvest almond flour package 2lb size"

**Poor Alt Text Examples:**
- "IMG_1234.jpg"
- "Cookie"
- "Gluten free gluten free gluten free food"
- "Image"

## FILE NAMING CONVENTIONS

### Naming Structure
Format: [category]-[description]-[modifier]-[size].extension
Examples:
- recipe-chocolate-chip-cookies-hero-1920x1080.jpg
- product-almond-flour-organic-harvest-800x800.webp
- blog-celiac-disease-guide-featured-1200x630.jpg
- instruction-mixing-bowl-step-3-600x400.jpg

### Naming Rules
- Use lowercase letters only
- Replace spaces with hyphens
- No special characters except hyphens
- Keep names under 50 characters
- Include relevant keywords naturally
- Be descriptive but concise
- Add size dimensions for variants
- Use logical categorization

## IMAGE FORMATS AND USAGE

### WebP (Preferred Format)
- Use for: All images where supported
- Benefits: 25-35% smaller than JPEG, supports transparency
- Quality setting: 85-90 for photos, 100 for graphics
- Browser support: 95%+ (with fallbacks)

### JPEG/JPG
- Use for: Photographs, complex images
- Quality setting: 85-90% for web
- Progressive encoding: Always enabled
- Color profile: sRGB
- Metadata: Strip EXIF data for privacy

### PNG
- Use for: Logos, icons, images needing transparency
- Compression: Maximum compression (PNG-8 when possible)
- Interlacing: Only for images above 10KB
- Color reduction: When quality permits

### AVIF (Next-Gen Format)
- Use for: Modern browsers, progressive enhancement
- Benefits: 50% smaller than JPEG
- Quality setting: 80-85
- Implementation: With WebP and JPEG fallbacks

### SVG
- Use for: Logos, icons, simple graphics
- Optimization: Minified and compressed
- Accessibility: Include title and desc elements
- Security: Sanitized to prevent XSS

## LAZY LOADING IMPLEMENTATION

### Native Lazy Loading
```html
<img src="recipe.jpg" 
     alt="Gluten-free bread" 
     loading="lazy"
     width="800" 
     height="600">
```

### Intersection Observer
- Load images 200px before viewport
- Placeholder blur-up technique
- Progressive image loading
- Low-quality image placeholder (LQIP)

### Critical Images
Never lazy load:
- Logo
- Hero images above fold
- First 3 images on page
- Navigation icons
- LCP (Largest Contentful Paint) images

## RESPONSIVE IMAGES

### Picture Element Usage
```html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <source media="(min-width: 1200px)" srcset="image-large.jpg">
  <source media="(min-width: 768px)" srcset="image-medium.jpg">
  <img src="image-small.jpg" alt="Description">
</picture>
```

### Srcset Implementation
```html
<img srcset="small.jpg 300w,
             medium.jpg 600w,
             large.jpg 1200w"
     sizes="(max-width: 300px) 100vw,
            (max-width: 600px) 50vw,
            33vw"
     src="medium.jpg"
     alt="Gluten-free recipe">
```
